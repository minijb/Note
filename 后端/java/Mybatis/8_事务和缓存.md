# 事务

多个操作同时完成或同时失败

特性：一致性，持久性，原子性，隔离性

以下订单为例:

1. 订单中完成增加一条记录
2. 订单明细表中完成N条记录的增加
3. 商品数据更新（减少）
4. 购物车中已支付商品删除
5. 用户积分更新

## 配置

设置事务

```xml
<environment id="development">
    <!--            配置事务控制交给
                            type:JDBC事务的控制交给传许愿 ，MANAGED：由容器(Spring)来管理事务-->
    <transactionManager type="JDBC"/>

</environment>
```

设置自动提交

```java
sqlsession = factory.openSession(true);
```
