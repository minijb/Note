# 工厂方法模式

## 1. 简单方法工厂

就是将具体的方法作为一个类，通过公共的接口别换不同的类实现不同的方法

# 2. 工厂方法模式

![image.png](https://s2.loli.net/2022/02/17/IGMRUQ8vfHsqo7p.png)

```java
Ifactory  operfactory = new AddFactory();
Operation oper = operfactory.createrOperation();
```

简单工厂模式的最大有点是工厂类包含了必要的条件判断，根据用户的需求动态的实例化相关的类，去除了和具体产品的历来，比如之间我们只需要把"+"给工厂，工厂就会返回具体的类，但是如果我们要增加新的方法就要修改工厂方法，这就违背了开放封闭原则

因此我们引入了工厂方法模式：定义一个用于创建对象的接口，让子类决定实例化哪一个类，工厂方法使一个类的实例化延迟到子类

![image.png](https://s2.loli.net/2022/02/17/6iHW83Df2So7NYA.png)

这样如果我们需要添加新的运算方法我们只需要添加新的工厂类就可以了，客户决定实现哪个工厂来实现类！

### 例子：雷锋工厂

```java
public class Leifen {
    public void Sweep(){
        System.out.print("Sweep");
    }

    public void Wash(){
        System.out.print("Wash");
    }
}

public class Undergraduate extends Leifen{
    
}

public class Volunteer extends Leifen{

}

public interface Ifactory {
    Leifen CreateLeifen();
}

public class UndergraduateFactory implements Ifactory{

    @Override
    public Leifen CreateLeifen() {
        return new Undergraduate();
    }
    

}

public class VolunteerFactory implements Ifactory {

    @Override
    public Leifen CreateLeifen() {
        return new Volunteer();
    }
    
}


public class App {
    public static void main(String[] args) throws Exception {
        Ifactory fac = new UndergraduateFactory();//我们只需要修改这个
        Leifen student = fac.CreateLeifen();
    }
}




```
